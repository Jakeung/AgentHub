# AgentHub 开发文档 - 总览

> 版本: v3.0（可开发版）  
> 日期: 2026-05-08

---

## 1. 项目定位

AgentHub 是一个 **Hermes Agent 实例管理平台**。

- **被管对象**: `nousresearch/hermes-agent` Docker 容器（每用户独立实例，容器内端口 8642）
- **核心价值**: 把手动 `docker run` + `.env` 文件管理 → Web 界面一站式管控
- **附加能力**: 内置 Web Chat（WS 代理到 Hermes 容器）、多渠道状态查看

## 2. 技术栈

| 层级 | 技术 | 版本 | 说明 |
|------|------|------|------|
| 前端框架 | Vue 3 + Vite | 3.5+ | Composition API + TypeScript |
| UI 组件 | ElementPlus | 2.x | 管理后台标准组件库 |
| 状态管理 | Pinia | 3.x | Vue 官方状态管理 |
| 后端框架 | FastAPI | 0.110+ | 异步、自动文档、Pydantic 校验 |
| ORM | SQLAlchemy 2.0 | 2.0+ | async 模式 + SQLite |
| 数据库 | SQLite | 3 | 嵌入式，零运维 |
| 容器管理 | docker-py | 7.x | Docker 官方 Python SDK |
| 认证 | JWT (python-jose) | — | HttpOnly Cookie |
| 部署 | Docker Compose | 3.8+ | AgentHub 自身也容器化 |

## 6. 统一约定

### 6.1 API 响应格式

```python
# 成功
{"code": 0, "data": {...}}

# 成功（列表 + 分页）
{"code": 0, "data": {"items": [...], "total": 100, "page": 1, "page_size": 20}}

# 失败
{"code": -1, "message": "参数错误"}
```

### 6.2 业务错误码

| code | 含义 | HTTP Status |
|------|------|-------------|
| 0 | 成功 | 200 |
| -1 | 参数错误 | 400 |
| -2 | 未认证 | 401 |
| -3 | 无权限 | 403 |
| -4 | 资源不存在 | 404 |
| -5 | 服务器错误 | 500 |
| -6 | 运行时依赖不可用 | 503 |

### 6.3 命名规范

| 位置 | 风格 | 示例 |
|------|------|------|
| 数据库表名 | snake_case | `agent_instance` |
| 数据库字段 | snake_case | `owner_user_id` |
| Python 变量/函数 | snake_case | `create_instance()` |
| Python 类名 | PascalCase | `AgentInstance` |
| API 路径 | kebab-case 或 snake_case | `/api/instances/{id}/model-config` |
| Vue 组件 | PascalCase | `InstanceList.vue` |
| Vue composable | camelCase | `useAuth.ts` |
| 权限码 | `module:action` | `instance:edit` |

## 7. 角色与页面分离

### 7.1 设计方案: 双布局切换

登录后根据角色自动跳转到不同布局，路由前缀区分：

| 路由前缀 | 布局 | 角色 | 页面 |
|----------|------|------|------|
| `/login` | 无 | 公开 | 登录/注册 |
| `/user/*` | UserLayout | user | 我的实例、对话、密钥 |
| `/admin/*` | AdminLayout | admin | 仪表盘、全局实例、用户管理、审计 |

### 7.2 两种角色对比

| 功能 | 普通用户 (user) | 管理员 (admin) |
|------|----------------|----------------|
| 登录/注册 | ✅ | ✅ |
| 实例管理 | 仅自己的，卡片布局，含创建/启停/删除 | 全局所有实例，表格布局，可强制停止/删除 |
| 对话 | 与自己的实例对话 | ❌ 不需要 |
| 密钥管理 | 管理自己的 API Key | ❌ 不需要 |
| 渠道配置 | 配置自己实例的渠道 | ❌ 不需要 |
| 用户管理 | ❌ | 管理所有用户（角色、状态） |
| 审计日志 | ❌ | 查看全局操作日志 |
| 系统监控 | ❌ | 仪表盘（CPU/内存/实例统计） |

### 7.3 路由表

```typescript
// /user/* — 普通用户路由组（UserLayout）
/user/instances          // 我的实例列表
/user/instances/:id      // 实例详情
/user/chat               // 对话页
/user/chat/:instanceId   // 与指定实例对话
/user/secrets            // 密钥管理

// /admin/* — 管理员路由组（AdminLayout，懒加载）
/admin/dashboard         // 系统仪表盘
/admin/instances         // 全局实例管理
/admin/users             // 用户管理
/admin/audit             // 审计日志
```

### 7.4 登录跳转逻辑

```typescript
// 登录成功后
if (user.role === 'admin') {
  router.push('/admin/dashboard')
} else {
  router.push('/user/instances')
}
```
