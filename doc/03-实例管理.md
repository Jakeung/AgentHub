# AgentHub - 实例管理

---

## 1. 业务概述

实例管理是 AgentHub 的核心模块。每个"实例"对应一个 `nousresearch/hermes-agent` Docker 容器。

---

## 2. 实例状态机

```
                    ┌─────────┐
         ┌─────────│ creating │
         │         └─────┬───┘
         │               │ docker create 成功
         │               ▼
         │         ┌──────────┐    docker start
         │    ┌────│ stopped  │──────────────┐
         │    │    └──────────┘              │
         │    │         ▲                    ▼
         │    │         │ stop 完成    ┌──────────┐
         │    │         │             │ starting │
         │    │         │             └─────┬────┘
         │    │    ┌──────────┐             │ 容器运行中
         │    │    │ stopping │             ▼
         │    │    └──────────┘       ┌──────────┐
         │    │         ▲             │ running  │────┐
         │    │         │ stop        └──────────┘    │
         │    │         │                  │           │ restart
         │    │         └──────────────────┘           │
         │    │                                        ▼
         │    │                               ┌────────────┐
         │    │                               │ restarting │
         │    │                               └────────────┘
         │    │
         │    │ delete                   ┌─────────┐
         │    └─────────────────────────→│ deleted │
         │                               └─────────┘
         │
         │    容器异常退出
         └────────────────────────────→ ┌─────────┐
                                        │  error  │
                                        └─────────┘
```

**状态转换表**:

| 当前状态 | 操作 | 目标状态 | 说明 |
|----------|------|----------|------|
| — | create | creating → stopped | 创建容器但不启动 |
| stopped | start | starting → running | 启动容器 |
| running | stop | stopping → stopped | 停止容器 |
| running | restart | restarting → running | 重启容器 |
| stopped | delete | deleted | 删除容器和数据目录 |
| error | start | starting → running | 从异常恢复 |
| error | delete | deleted | 删除异常实例 |
| * | (容器崩溃) | error | Docker 事件检测 |

## 3. Docker 操作映射

每个 API 操作对应的 docker-py 调用:

| 操作 | docker-py 调用 | 参数 |
|------|----------------|------|
| 创建 | `client.containers.create()` | image, name, ports, volumes, mem_limit, cpus, environment, command |
| 启动 | `container.start()` | — |
| 停止 | `container.stop(timeout=10)` | 10s 优雅关闭 |
| 重启 | `container.restart(timeout=10)` | — |
| 删除 | `container.remove(force=True)` | 强制删除 |
| 日志 | `container.logs(stream=True, follow=True)` | 流式日志 |
| 状态 | `container.status` / `container.attrs` | 容器详情 |
| 资源 | `container.stats(stream=False)` | CPU/内存使用 |

**创建容器的完整参数**:

```python
container = docker_client.containers.create(
    image="nousresearch/hermes-agent:latest",
    name=f"hermes-{instance.container_name}",
    command="gateway run",
    ports={"8642/tcp": instance.port},
    volumes={instance.data_dir: {"bind": "/opt/data", "mode": "rw"}},
    mem_limit=f"{instance.memory_limit_mb}m",
    nano_cpus=int(instance.cpu_limit * 1e9),
    environment=build_env_dict(instance),  # 从 env_config JSON 构建
    restart_policy={"Name": "unless-stopped"},
    detach=True,
)
```

## 4. 端口分配策略

```python
PORT_RANGE_START = 9001
PORT_RANGE_END = 9100

def allocate_port() -> int:
    """分配一个未被占用的端口"""
    used_ports = {inst.port for inst in db.query(AgentInstance).filter(
        AgentInstance.status != 'deleted'
    )}
    for port in range(PORT_RANGE_START, PORT_RANGE_END + 1):
        if port not in used_ports:
            return port
    raise BusinessError("端口已耗尽，无法创建新实例")
```

## 5. API 接口定义

### 5.1 GET /api/instances

获取当前用户的实例列表。

**查询参数**:

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| status | string | — | 按状态过滤 |
| page | int | 1 | 页码 |
| page_size | int | 20 | 每页数量 |

**成功响应**:

```json
{
  "code": 0,
  "data": {
    "items": [
      {
        "id": 1,
        "name": "我的助手",
        "container_name": "hermes-alice-001",
        "port": 9001,
        "status": "running",
        "health_status": "healthy",
        "cpu_limit": 1.0,
        "memory_limit_mb": 2048,
        "created_at": "2026-05-08T10:00:00",
        "updated_at": "2026-05-08T10:05:00"
      }
    ],
    "total": 1,
    "page": 1,
    "page_size": 20
  }
}
```

### 5.2 POST /api/instances

创建新实例。

**请求**:

```json
{
  "name": "我的助手",
  "cpu_limit": 1.0,
  "memory_limit_mb": 2048,
  "env_config": {
    "DEEPSEEK_API_KEY": "sk-xxx",
    "WEIXIN_DM_POLICY": "open",
    "GATEWAY_ALLOW_ALL_USERS": "true"
  }
}
```

**校验规则**:

| 字段 | 规则 |
|------|------|
| name | 1-100 字符 |
| cpu_limit | 0.5 - 4.0 |
| memory_limit_mb | 512 - 8192 |
| env_config | 可选，JSON 对象 |

**服务端处理流程**:

```
1. 分配端口号
2. 生成 container_name: hermes-{username}-{seq}
3. 创建数据目录: /opt/hermes/{container_name}
4. 写入数据库 (status=creating)
5. 调用 docker create
6. 更新 status=stopped
7. 返回实例详情
```

**成功响应** (200):

```json
{
  "code": 0,
  "data": {
    "id": 2,
    "name": "我的助手",
    "container_name": "hermes-alice-001",
    "port": 9002,
    "status": "stopped",
    "cpu_limit": 1.0,
    "memory_limit_mb": 2048,
    "created_at": "2026-05-08T11:00:00"
  }
}
```

### 5.3 GET /api/instances/{id}

获取实例详情（含模型配置和渠道状态）。

**成功响应**:

```json
{
  "code": 0,
  "data": {
    "id": 1,
    "name": "我的助手",
    "container_name": "hermes-alice-001",
    "port": 9001,
    "status": "running",
    "health_status": "healthy",
    "cpu_limit": 1.0,
    "memory_limit_mb": 2048,
    "data_dir": "/opt/hermes/hermes-alice-001",
    "env_config": {
      "DEEPSEEK_API_KEY": "sk-****xxxx",
      "WEIXIN_DM_POLICY": "open"
    },
    "model_config": {
      "provider": "deepseek",
      "model_name": "deepseek-chat"
    },
    "channels": [
      { "platform": "weixin", "status": "connected" }
    ],
    "created_at": "2026-05-08T10:00:00",
    "updated_at": "2026-05-08T10:05:00"
  }
}
```

> 注意: `env_config` 中的敏感值（API Key）返回时脱敏为掩码。

### 5.4 PUT /api/instances/{id}

更新实例配置。

**请求**:

```json
{
  "name": "新名称",
  "cpu_limit": 2.0,
  "memory_limit_mb": 4096,
  "env_config": {
    "DEEPSEEK_API_KEY": "sk-newkey",
    "WEIXIN_DM_POLICY": "open"
  }
}
```

> 如果实例正在运行，资源配置变更需要先停止再启动（提示用户）。
> `env_config` 变更在下次启动时生效。

### 5.5 DELETE /api/instances/{id}

删除实例（容器 + 数据目录）。

**前置条件**: 实例状态必须为 `stopped` 或 `error`。

**成功响应**:

```json
{ "code": 0, "data": null }
```

**失败**:

| code | message |
|------|---------|
| -1 | 请先停止实例再删除 |

### 5.6 POST /api/instances/{id}/start

启动实例。

**前置条件**: 状态为 `stopped` 或 `error`。

**成功响应**:

```json
{
  "code": 0,
  "data": { "status": "starting" }
}
```

### 5.7 POST /api/instances/{id}/stop

停止实例。

**前置条件**: 状态为 `running`。

**成功响应**:

```json
{
  "code": 0,
  "data": { "status": "stopping" }
}
```

### 5.8 POST /api/instances/{id}/restart

重启实例。

**前置条件**: 状态为 `running`。

### 5.9 GET /api/instances/{id}/logs

获取实例运行日志。

**查询参数**:

| 参数 | 类型 | 默认 | 说明 |
|------|------|------|------|
| tail | int | 100 | 最近 N 行 |
| since | string | — | ISO 时间起点 |
| level | string | — | 日志级别过滤 |

**成功响应**:

```json
{
  "code": 0,
  "data": {
    "logs": [
      {"time": "2026-05-08T10:01:00", "level": "info", "message": "Gateway started"},
      {"time": "2026-05-08T10:01:01", "level": "info", "message": "Connected to DeepSeek API"}
    ]
  }
}
```

### 5.10 GET /api/instances/{id}/stats

获取实例资源使用情况（CPU/内存）。

**成功响应**:

```json
{
  "code": 0,
  "data": {
    "cpu_percent": 15.2,
    "memory_used_mb": 512,
    "memory_limit_mb": 2048,
    "memory_percent": 25.0,
    "network_rx_bytes": 1048576,
    "network_tx_bytes": 524288
  }
}
```

## 6. 前端页面设计

> 实例管理存在两个完全不同的前端视角：普通用户管理自己的实例，管理员管理全局所有实例。

```

│  实例管理（全局）                                                    │

├───────┬──────────┬────────┬──────┬──────┬────────┬──────┬─────────┤
```

**特点**: 表格布局，显示所有用户的实例，含「所属用户」列，支持按用户筛选，admin 可强制停止/删除任意实例。

**组件结构**:

```
views/admin/instances/
├── index.vue              # 全局实例表格（el-table）
└── InstanceDetailDialog.vue # 实例详情弹窗（只读 + 操作按钮）
```

**API 差异**:

| | 用户端 | 管理端 |
|---|---|---|
| API | `GET /api/instances` | `GET /api/admin/instances` |
| 数据范围 | 自动过滤 `owner_user_id` | 返回全部，可选 `user_id` 筛选 |
| 操作 | 完整 CRUD + 启停 | 仅查看 + 强制停止/删除 |
| 创建 | ✅ | ❌ admin 不创建实例 |

## 7. 管理端 API（admin 专属）

### 7.1 GET /api/admin/instances

获取全局所有用户的实例列表。

**权限**: `admin:view`

**查询参数**:

| 参数 | 类型 | 说明 |
|------|------|------|
| user_id | int | 按所属用户过滤 |
| status | string | 按状态过滤 |
| page | int | 页码 |
| page_size | int | 每页数量 |

**成功响应**:

```json
{
  "code": 0,
  "data": {
    "items": [
      {
        "id": 1,
        "name": "我的助手",
        "container_name": "hermes-alice-001",
        "port": 9001,
        "status": "running",
        "owner_user_id": 2,
        "owner_username": "alice",
        "cpu_limit": 1.0,
        "memory_limit_mb": 2048,
        "created_at": "2026-05-08T10:00:00"
      }
    ],
    "total": 10,
    "page": 1,
    "page_size": 20
  }
}
```

### 7.2 POST /api/admin/instances/{id}/force-stop

强制停止任意实例。

**权限**: `admin:edit`

### 7.3 DELETE /api/admin/instances/{id}

强制删除任意实例（含容器 + 数据）。

**权限**: `admin:edit`
