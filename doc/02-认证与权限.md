# AgentHub - 认证与权限

---

## 1. 业务流程

### 1.1 注册流程（仅创建普通用户）

> 注册接口 **只能创建普通用户角色**。管理员账号仅通过以下方式创建：
> 1. 系统初始化时自动创建默认 admin 账号
> 2. 已有 admin 在管理后台「用户管理」页面新建管理员

```
用户 → POST /api/auth/register { username, email, password }
  ├─ Pydantic 校验（用户名 3-50 字符，密码 ≥ 6 位）
  ├─ 检查 username / email 唯一性
  ├─ bcrypt 加密密码
  ├─ 创建用户（默认 role_id = 2，即 user 角色）
  ├─ 记录审计日志
  └─ 返回 { id, username, email }
```

### 1.2 登录流程

```
用户 → POST /api/auth/login { username, password }
  ├─ 速率限制检查（同 IP 15 分钟内最多 10 次）
  ├─ 查询用户（含角色 + 权限列表）
  ├─ bcrypt 比对密码
  ├─ 签发 JWT（payload: id, username, role, permissions）
  ├─ 设置 HttpOnly Cookie（auth-token）
  ├─ 记录审计日志（login / login_failed）
  └─ 返回 { user, permissions }
```

### 1.3 认证中间件流程

```
每个请求 → Auth Middleware
  ├─ 公开路径放行（/api/auth/login, /api/auth/register）
  ├─ 从 Cookie 读取 auth-token
  ├─ JWT 验证签名 + 过期时间
  ├─ 将 user_id / role / permissions 注入 request.state
  └─ 业务层通过 request.state 做权限判断
```

### 1.4 权限检查流程

```
API Handler
  ├─ 调用 check_permission(request, "instance:edit")
  ├─ 从 request.state.permissions 查找
  ├─ 通过 → 执行业务逻辑
  └─ 拒绝 → 返回 { code: -3, message: "无权限" }
```

## 2. RBAC 权限模型

### 2.1 预置角色

| 角色 | 权限范围 |
|------|----------|
| admin | 全部 12 个权限 |
| user | instance:view/edit/delete, chat:use, channel:view/edit, apikey:view/edit |

### 2.2 权限码清单

| 权限码 | 名称 | 模块 | admin | user |
|--------|------|------|:-----:|:----:|
| `instance:view` | 查看实例 | instance | ✅ | ✅ |
| `instance:edit` | 编辑实例 | instance | ✅ | ✅ |
| `instance:delete` | 删除实例 | instance | ✅ | ✅ |
| `chat:use` | 使用对话 | chat | ✅ | ✅ |
| `channel:view` | 查看渠道 | channel | ✅ | ✅ |
| `channel:edit` | 编辑渠道 | channel | ✅ | ✅ |
| `apikey:view` | 查看密钥 | apikey | ✅ | ✅ |
| `apikey:edit` | 编辑密钥 | apikey | ✅ | ✅ |
| `admin:view` | 查看管理 | admin | ✅ | ❌ |
| `admin:edit` | 编辑管理 | admin | ✅ | ❌ |
| `audit:view` | 查看审计 | audit | ✅ | ❌ |
| `monitor:view` | 查看监控 | monitor | ✅ | ❌ |

### 2.3 管理员账号创建规则

- 注册 API (`POST /api/auth/register`) **硬编码** role_id = 2（user），无法通过参数指定 admin
- admin 账号仅能通过：
  1. 系统初始化 SQL（`01-数据库设计.md` 中的 admin 用户）
  2. 管理后台 `POST /api/admin/users`（可指定 role_id = 1）
- 只有 admin 角色有权调用 `/api/admin/users` 创建新用户

### 2.4 租户隔离规则

- 普通用户所有查询自动附加 `WHERE owner_user_id = current_user_id`
- admin 角色不附加，可查看全局数据
- 判断方式: `request.state.role == 'admin'`

## 3. JWT 设计

| 项 | 值 |
|-----|-----|
| 算法 | HS256 |
| 有效期 | 24 小时 |
| 存储 | HttpOnly Cookie (`auth-token`) |
| Secure | 生产环境 true |
| SameSite | Lax |
| 密钥来源 | 环境变量 `JWT_SECRET` |

**Payload 结构**:

```json
{
  "id": 1,
  "username": "admin",
  "role": "admin",
  "permissions": ["instance:view", "instance:edit", "..."],
  "exp": 1720000000
}
```

## 4. API 接口定义

### 4.1 POST /api/auth/register

注册新用户。

**请求**:

```json
{
  "username": "alice",
  "email": "alice@example.com",
  "password": "mypassword123"
}
```

**校验规则**:

| 字段 | 规则 |
|------|------|
| username | 3-50 字符，`^[a-zA-Z0-9_]+$` |
| email | 合法邮箱，可选 |
| password | ≥ 6 字符 |

**成功响应** (200):

```json
{
  "code": 0,
  "data": {
    "id": 2,
    "username": "alice",
    "email": "alice@example.com"
  }
}
```

**失败响应**:

| code | message | 场景 |
|------|---------|------|
| -1 | 用户名已存在 | username 重复 |
| -1 | 邮箱已被注册 | email 重复 |
| -1 | 密码至少6个字符 | 校验不通过 |

### 4.2 POST /api/auth/login

登录并获取 Token。

**请求**:

```json
{
  "username": "alice",
  "password": "mypassword123"
}
```

**成功响应** (200):

```json
{
  "code": 0,
  "data": {
    "user": {
      "id": 2,
      "username": "alice",
      "name": "Alice",
      "role": "user"
    },
    "permissions": ["instance:view", "instance:edit", "..."]
  }
}
```

> 同时设置 Cookie: `auth-token=<JWT>; HttpOnly; Path=/; SameSite=Lax`

**失败响应**:

| code | message | 场景 |
|------|---------|------|
| -1 | 用户名或密码错误 | 认证失败 |
| -5 | 登录尝试过于频繁 | 速率限制 |

### 4.3 POST /api/auth/logout

登出（清除 Cookie）。

**成功响应** (200):

```json
{ "code": 0, "data": null }
```

> 设置 Cookie: `auth-token=; HttpOnly; Path=/; Max-Age=0`

### 4.4 GET /api/auth/me

获取当前登录用户信息。

**成功响应** (200):

```json
{
  "code": 0,
  "data": {
    "id": 2,
    "username": "alice",
    "name": "Alice",
    "email": "alice@example.com",
    "role": "user",
    "permissions": ["instance:view", "..."],
    "last_login_at": "2026-05-08T10:30:00"
  }
}
```

**失败响应**:

| code | message | 场景 |
|------|---------|------|
| -2 | 未登录 | 无 Token 或 Token 过期 |

**组件结构**:

```
views/login/
├── index.vue           # 登录页主组件
└── RegisterDialog.vue  # 注册弹窗（el-dialog）
```

**交互逻辑**:

1. 登录成功 → 存储用户信息到 Pinia store → 按角色跳转:

### 5.3 路由守卫

```typescript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()

  // 公开页面
  if (to.path === '/login') return next()

  // 未登录
  if (!userStore.isLoggedIn) return next('/login')

  // admin 访问 /user/* → 重定向到 /admin/dashboard
  if (to.path.startsWith('/user') && userStore.role === 'admin') {
    return next('/admin/dashboard')
  }

  // user 访问 /admin/* → 403
  if (to.path.startsWith('/admin') && userStore.role !== 'admin') {
    return next('/403')
  }

  next()
})
```
