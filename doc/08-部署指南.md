# AgentHub 部署指南

## 📋 项目概览

**AgentHub** 是一个全栈应用，包含：
- **后端**: FastAPI + SQLite + Python 3.x
- **前端**: Vue 3 + TypeScript + Vite
- **容器化**: Docker + Docker Compose

**规模**: 小型应用 (<100 用户，<10 并发)
**目标**: 易维护、可靠、支持多轮迭代更新

---

## 🚀 快速开始

### 前置要求
- 腾讯云 Ubuntu 20.04+ 或本地 Linux/WSL2
- Docker 20.10+
- Docker Compose 2.0+
- Git

### 安装 Docker（如还未安装）

```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker --version
docker-compose --version
```

## 📦 生产环境部署（腾讯云 Ubuntu）

### Step 1: 克隆代码

```bash
# 在合适的目录创建部署文件夹
mkdir -p ~/agenthub
cd ~/agenthub

# 克隆项目（假设已有 git 权限）
git clone <your-repo-url> .

# 或者上传项目文件
```

### Step 2: 配置环境变量

```bash
# 复制示例文件
cp .env.example .env

# 编辑 .env（必须修改 JWT_SECRET）
nano .env
```

**必需修改的配置**:
```env
# 生成安全的 JWT_SECRET
# 在服务器上运行: openssl rand -hex 32
JWT_SECRET=<output-of-openssl-command>

# 根据服务器 IP 配置
CORS_ORIGINS=http://<your-server-ip>

# 如果有域名后期可以改为: https://yourdomain.com
```

### Step 3: 一键部署

```bash
# 赋予执行权限
chmod +x deploy.sh

# 执行部署脚本
./deploy.sh prod

# 脚本会自动：
# ✓ 检查 Docker 环境
# ✓ 创建数据目录
# ✓ 拉取最新代码
# ✓ 构建容器镜像
# ✓ 启动服务
# ✓ 检查服务健康状态
```

### Step 4: 验证部署

部署完成后，你会看到：
```
后端地址: http://<your-ip>:8000
前端地址: http://<your-ip>:80
```

在浏览器打开前端地址验证应用正常运行。

## 🛠️ 日常维护命令

### 查看日志
```bash
# 实时日志（按 Ctrl+C 退出）
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
docker-compose logs -f frontend
```

### 重启服务
```bash
# 重启所有服务
docker-compose restart

# 重启特定服务
docker-compose restart backend
docker-compose restart frontend
```

### 停止服务
```bash
docker-compose down
```

### 清理空间（删除旧镜像）
```bash
./deploy.sh prod cleanup
# 或
docker system prune -f
```

## 🔄 更新应用

### 方案 1: 使用部署脚本（推荐）

```bash
# 拉取最新代码 + 重新部署
./deploy.sh prod

# 脚本会：
# 1. 拉取最新 Git 代码
# 2. 重新构建镜像
# 3. 重启容器（自动 0-downtime）
```

### 方案 2: 手动更新代码

```bash
# 1. 停止服务
docker-compose down

# 2. 拉取代码
git pull origin main

# 3. 重新启动
docker-compose up -d
```

## 💾 数据备份

SQLite 数据库存储在 `./data/agenthub.db`

### 备份数据
```bash
# 复制数据文件到备份位置
cp -r ./data ./data_backup_$(date +%Y%m%d_%H%M%S)

# 或定期备份到其他服务器
rsync -avz ./data/ backup_server:/backup/agenthub/
```

### 恢复数据
```bash
# 停止服务
docker-compose down

# 恢复数据文件
cp -r ./data_backup_YYYYMMDD_HHMMSS/* ./data/

# 重启服务
docker-compose up -d
```

## 🔐 安全建议

### 生产环境检查清单

- [ ] JWT_SECRET 已修改为随机字符串（不用默认值）
- [ ] CORS_ORIGINS 已限制为实际域名/IP
- [ ] 防火墙规则已配置（只开放 80 和 8000 端口）
- [ ] 定期备份数据库
- [ ] 监控磁盘使用（SQLite 日志会占用空间）

### 防火墙配置（UFW）

```bash
# 安装 UFW（如未安装）
sudo apt install ufw

# 允许 SSH
sudo ufw allow 22/tcp

# 允许 HTTP
sudo ufw allow 80/tcp

# 允许后端 API
sudo ufw allow 8000/tcp

# 启用防火墙
sudo ufw enable
```

## 📊 监控和故障排查

### 检查服务状态

```bash
# 查看运行中的容器
docker ps

# 查看容器详细信息
docker-compose ps

# 查看具体日志错误
docker-compose logs backend | tail -50
```

### 常见问题

**Q: 后端无法连接？**
```bash
# 检查后端日志
docker-compose logs backend

# 重启后端
docker-compose restart backend
```

**Q: 前端显示无法连接 API？**
- 检查 CORS_ORIGINS 配置是否正确
- 确认后端服务正在运行

**Q: 磁盘空间不足？**
```bash
# 清理旧 Docker 镜像和数据
./deploy.sh prod cleanup

# 检查数据库大小
du -sh ./data/
```

## 📈 未来扩展

当应用规模扩大时的升级路径：

### 从 SQLite 升级到 MySQL
1. 修改 `DATABASE_URL` 为 `mysql://...`
2. 迁移历史数据
3. 添加 MySQL 容器到 docker-compose.yml

### 添加 HTTPS/域名
1. 配置 Nginx 反向代理
2. 使用 Let's Encrypt 免费证书
3. 配置 CORS_ORIGINS 为 https://yourdomain.com

### 高可用部署
1. 迁移到 Kubernetes（如需要）
2. 配置多实例负载均衡
3. 实现数据库主从复制
