# AgentHub - 安全与规范

---

## 1. 安全基线检查清单

### 1.1 认证安全

| 项 | 要求 | 实现方式 |
|----|------|----------|
| 密码加密 | bcrypt, cost ≥ 10 | `bcrypt.hashpw(password, bcrypt.gensalt(10))` |
| JWT 存储 | HttpOnly Cookie | 不使用 localStorage |
| JWT 有效期 | 24h | `JWT_EXPIRE_HOURS=24` |
| Cookie Secure | 生产环境开启 | `secure=True` if HTTPS |
| SameSite | Lax | 防止 CSRF |
| 登录速率限制 | 同 IP 15 分钟 ≤ 10 次 | 内存计数器 + 滑动窗口 |

### 1.2 数据安全

| 项 | 要求 | 实现方式 |
|----|------|----------|
| API 密钥存储 | AES-256-GCM 加密 | `app/core/encryption.py` |
| 密钥传输 | 仅创建时明文传入 | 读取时返回掩码 `sk-****xxxx` |
| 加密密钥管理 | 环境变量 | `AES_SECRET_KEY` |
| JWT 签名密钥 | 环境变量 | `JWT_SECRET` ≥ 32 字符随机串 |
| 数据库文件权限 | 600 | 仅应用进程可读写 |

### 1.3 输入校验

| 项 | 要求 |
|----|------|
| 所有入参 | Pydantic model 校验 |
| SQL 注入 | SQLAlchemy ORM，不拼接 SQL |
| XSS | 前端不使用 v-html 渲染用户输入 |
| 路径穿越 | data_dir 仅允许 `/opt/hermes/` 前缀 |
| 容器名注入 | `container_name` 仅允许 `^[a-z0-9-]+$` |

### 1.4 网络安全

| 项 | 要求 |
|----|------|
| CORS | 仅允许前端域名 |
| Hermes 容器端口 | 不对外暴露，仅绑定 127.0.0.1 |
| 管理端口 | 8000 通过 Nginx 反代 |
| Docker Socket | 挂载到 AgentHub 容器 |

## 2. 统一 API 响应格式

### 2.1 响应结构

```typescript
interface ApiResponse<T> {
  code: number    // 0=成功, 负数=错误
  data: T | null  // 业务数据
  message?: string // 错误时的提示信息
}
```

### 2.2 分页响应

```typescript
interface PageResponse<T> {
  items: T[]
  total: number
  page: number
  page_size: number
}
```

### 2.3 后端统一封装

```python
from fastapi.responses import JSONResponse

def success(data=None):
    return {"code": 0, "data": data}

def error(code: int, message: str):
    return {"code": code, "message": message, "data": None}

def page_data(items, total, page, page_size):
    return success({
        "items": items,
        "total": total,
        "page": page,
        "page_size": page_size,
    })
```

## 3. 错误码矩阵

| code | 含义 | 使用场景 |
|------|------|----------|
| 0 | 成功 | 所有成功响应 |
| -1 | 业务错误 | 参数校验失败、业务规则不满足 |
| -2 | 未认证 | 未登录、Token 过期 |
| -3 | 无权限 | 权限不足 |
| -4 | 资源不存在 | ID 不存在、已删除 |
| -5 | 速率限制 | 请求过于频繁 |
| -10 | Docker 错误 | 容器操作失败 |
| -11 | 端口耗尽 | 无可用端口 |
| -99 | 系统内部错误 | 未预期的异常 |

### 3.1 全局异常处理

```python
from fastapi import Request
from fastapi.responses import JSONResponse

class BusinessError(Exception):
    def __init__(self, code: int = -1, message: str = ""):
        self.code = code
        self.message = message

@app.exception_handler(BusinessError)
async def business_error_handler(request: Request, exc: BusinessError):
    return JSONResponse(
        status_code=200,
        content={"code": exc.code, "message": exc.message, "data": None}
    )

@app.exception_handler(Exception)
async def global_error_handler(request: Request, exc: Exception):
    logger.error(f"Unhandled error: {exc}", exc_info=True)
    return JSONResponse(
        status_code=500,
        content={"code": -99, "message": "系统内部错误", "data": None}
    )
```

## 4. 命名规范

### 4.1 后端

| 类别 | 规范 | 示例 |
|------|------|------|
| 文件名 | snake_case | `instance_service.py` |
| 类名 | PascalCase | `InstanceService` |
| 函数名 | snake_case | `create_instance()` |
| 常量 | UPPER_SNAKE | `PORT_RANGE_START` |
| 数据库表 | snake_case | `agent_instance` |
| 数据库列 | snake_case | `owner_user_id` |
| API 路径 | kebab-case 或名词 | `/api/instances/{id}/model-config` |

### 4.2 前端

| 类别 | 规范 | 示例 |
|------|------|------|
| 组件文件 | PascalCase | `InstanceCard.vue` |
| 组合式函数 | camelCase，use 前缀 | `useInstanceStore.ts` |
| API 模块 | camelCase | `instanceApi.ts` |
| CSS 类名 | BEM 或 kebab-case | `instance-card__status` |
| 路由名 | kebab-case | `instance-detail` |
| Store 名 | camelCase，use 前缀 | `useUserStore` |

## 5. 前端 API 封装规范

```typescript
// api/request.ts - Axios 统一封装
import axios from 'axios'
import { ElMessage } from 'element-plus'
import { useUserStore } from '@/stores/user'
import router from '@/router'

const request = axios.create({
  baseURL: '/api',
  timeout: 30000,
  withCredentials: true,  // Cookie 自动携带
})

request.interceptors.response.use(
  (response) => {
    const { code, message } = response.data
    if (code === 0) return response.data
    if (code === -2) {
      useUserStore().logout()
      router.push('/login')
    }
    ElMessage.error(message || '请求失败')
    return Promise.reject(response.data)
  },
  (error) => {
    ElMessage.error('网络错误')
    return Promise.reject(error)
  }
)

export default request
```

```typescript
// api/instance.ts - 实例模块 API
import request from './request'

export const instanceApi = {
  list: (params?: { status?: string; page?: number }) =>
    request.get('/instances', { params }),
  detail: (id: number) =>
    request.get(`/instances/${id}`),
  create: (data: CreateInstanceReq) =>
    request.post('/instances', data),
  update: (id: number, data: UpdateInstanceReq) =>
    request.put(`/instances/${id}`, data),
  delete: (id: number) =>
    request.delete(`/instances/${id}`),
  start: (id: number) =>
    request.post(`/instances/${id}/start`),
  stop: (id: number) =>
    request.post(`/instances/${id}/stop`),
  restart: (id: number) =>
    request.post(`/instances/${id}/restart`),
  logs: (id: number, params?: { tail?: number }) =>
    request.get(`/instances/${id}/logs`, { params }),
  stats: (id: number) =>
    request.get(`/instances/${id}/stats`),
}
```

## 6. 环境变量清单

| 变量 | 必需 | 说明 | 示例 |
|------|------|------|------|
| `JWT_SECRET` | 是 | JWT 签名密钥 | 32+ 字符随机串 |
| `AES_SECRET_KEY` | 是 | API 密钥加密 | 32 字节 Base64 |
| `DATABASE_URL` | 否 | 数据库路径 | `sqlite+aiosqlite:///data/agenthub.db` |
| `CORS_ORIGINS` | 否 | 允许的前端域名 | `http://localhost:5173` |
| `LOG_LEVEL` | 否 | 日志级别 | `INFO` |
| `HERMES_DATA_ROOT` | 否 | 实例数据根目录 | `/opt/hermes` |
| `PORT_RANGE_START` | 否 | 端口范围起始 | `9001` |
| `PORT_RANGE_END` | 否 | 端口范围结束 | `9100` |

## 7. Docker Compose 部署配置

```yaml
version: "3.8"

services:
  agenthub-backend:
    build: ./backend
    ports:
      - "8000:8000"
    volumes:
      - ./data:/app/data                    # SQLite + 应用数据
      - /var/run/docker.sock:/var/run/docker.sock  # Docker 控制
      - /opt/hermes:/opt/hermes             # 实例数据目录
    environment:
      - JWT_SECRET=${JWT_SECRET}
      - AES_SECRET_KEY=${AES_SECRET_KEY}
      - CORS_ORIGINS=http://localhost:5173
    restart: unless-stopped

  agenthub-web:
    build: ./web
    ports:
      - "5173:80"
    depends_on:
      - agenthub-backend
    restart: unless-stopped
```
