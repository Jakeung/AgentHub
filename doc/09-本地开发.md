# AgentHub 本地开发指南

快速开始在本地进行开发。支持 Windows WSL2、Linux 和 macOS。

---

## 🔧 前置要求

- Node.js 18+ (前端)
- Python 3.9+ (后端)
- Docker + Docker Compose (可选，用于数据库等)
- Git

### 快速检查环境

```bash
node --version      # 需要 18+
npm --version       # 需要 8+
python --version    # 需要 3.9+
```

---

## 🚀 快速启动 (推荐: Docker Compose)

如果已安装 Docker，最简单的方式：

```bash
# 1. 进入项目目录
cd AgentHub

# 2. 复制环境配置
cp .env.example .env

# 3. 一键启动所有服务
docker-compose up

# 4. 打开浏览器
# 前端: http://localhost:80
# 后端 API: http://localhost:8000
# API 文档: http://localhost:8000/docs
```

停止服务: `Ctrl+C` 然后 `docker-compose down`

## 💻 手动启动 (更灵活的开发方式)

如果想分别运行后端和前端，以便更方便地调试：

### 后端启动

```bash
# 1. 进入后端目录
cd backend

# 2. 创建虚拟环境
python -m venv venv

# 3. 激活虚拟环境
# Linux/macOS:
source venv/bin/activate

# Windows (PowerShell):
.\venv\Scripts\Activate.ps1

# Windows (CMD):
venv\Scripts\activate.bat

# 4. 安装依赖
pip install -r requirements.txt

# 5. 创建数据目录
mkdir -p data

# 6. 启动后端服务
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

# ✓ 后端已启动: http://localhost:8000
# ✓ API 文档: http://localhost:8000/docs (Swagger)
```

### 前端启动 (在新终端窗口)

```bash
# 1. 进入前端目录
cd web

# 2. 安装依赖
npm install

# 3. 启动开发服务器
npm run dev

# ✓ 前端已启动: http://localhost:5173 (或其他端口)
```

### 环境变量配置

在 `backend/.env` 中配置后端环境变量：

```env
# 开发环境不需要改这些，使用默认值即可
JWT_SECRET=dev-secret-key
DATABASE_URL=sqlite+aiosqlite:///data/agenthub.db
CORS_ORIGINS=http://localhost,http://localhost:5173
LOG_LEVEL=DEBUG
```

## 🐛 调试技巧

### 后端调试

1. **查看 API 文档**
   - 打开 http://localhost:8000/docs (Swagger UI)
   - 可以在线测试所有 API

2. **查看请求日志**
   - 在终端中查看所有请求和响应

3. **数据库检查**
   ```bash
   # 查看 SQLite 数据库内容
   sqlite3 data/agenthub.db
   sqlite> .tables          # 查看所有表
   sqlite> SELECT * FROM users;  # 查询数据
   sqlite> .quit            # 退出
   ```

### 前端调试

1. **浏览器开发者工具**
   - F12 打开开发者工具
   - 查看 Console、Network、Elements 选项卡

2. **Vue DevTools**
   - 安装 Vue DevTools 浏览器扩展
   - 检查组件状态和 Pinia store

3. **Network 检查**
   - 在 Network 选项卡查看 API 请求
   - 检查 CORS 错误、响应状态等

## 💡 开发最佳实践

1. **代码分离**
   - 逻辑放在 services/ 中
   - API 路由保持简洁
   - 可重用代码放在 core/ 或 utils/

2. **环境隔离**
   - 后端使用虚拟环境
   - 前端使用 node_modules
   - 不要提交本地配置

3. **提交前检查**
   - 运行格式化工具（black、prettier）
   - 确保没有调试代码（console.log、print）
   - 更新 .env.example（如有新的环境变量）

## 🆘 常见问题

**Q: 后端无法启动，报错 "port already in use"?**
```bash
# 查找占用 8000 端口的进程
lsof -i :8000

# 杀掉进程
kill -9 <PID>
```

**Q: 前端无法连接到后端 API?**
- 检查后端是否正在运行: http://localhost:8000/docs
- 检查 CORS_ORIGINS 配置（应包含前端地址）
- 查看浏览器控制台 (F12) 的错误信息

**Q: Python 虚拟环境激活失败?**
```bash
# 重新创建虚拟环境
rm -rf backend/venv
cd backend
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

**Q: 前端依赖安装失败?**
```bash
# 清除缓存重新安装
cd web
rm -rf node_modules package-lock.json
npm install
```
